PRFAQ 参照

エレベーターピッチ

不動産業界では、申請書類や契約書が規制やチェックリストに適合しているかを確認する作業に膨大な時間がかかり、人手不足や長時間労働の原因となっています。BEACON は生成 AI を活用して書類内の図面やテキストを自動分析し、適合性チェックを数分で完了させることができます。技術に詳しくなくても簡単に使え、即座に効果を体感できるため、不動産業務の効率化に劇的な変化をもたらします。AWS 上で稼働し、パートナー企業が顧客特有のニーズに合わせてカスタマイズできる柔軟性も備えています。

誰のためのプロジェクト？

- For Internal
  - アカウントチームが LoB 開拓のキッカケとなる
  - Azure / OpenAI から Winback を狙える
- For Customer
  - 業務効率化（残業抑制、人手不足など）において、Non-tech な人でもサクッと効果が体感できる
    - 「これでいいじゃん」感。「こここうすればいけるんじゃない？」感
  - PoC がこのアセットで完結する
    - PoC の定義
      - パートナーに解決してほしい課題が明確になっている
      - 課題
        - 精度
        - オペレーションとのギャップ
- For Partner
  - SI の土台となる
  - 営業の商材となる
    - 彼らのお客さんと会話するきっかけになる
    - （不動産未着手の場合）新しい業界の開拓

プロジェクトの成功の定義

- BEACON がきっかけで AWS が LoB を開拓でき、Revenue が立つこと
- 不動産業界において、市場で一定の認知を獲得すること

プロジェクトの失敗

- LoB の人に興味を持ってもらえない
  - 「うち困ってないし」
- パートナーが見つからない

やらないことリスト

- 細かな業務フローへの対応
  - たとえば申請チェックリストのフローチャートデザイン機能など
  - パートナーを開拓し、彼らの SI に任せる

プロジェクトのリスク・懸念点は何か？

技術的な問題点

- 書類に含まれる図面、テキスト、表などの非構造化データを、正しくセグメンテーションし、チェックリストに突合し合ってるか確認できるかどうか検証が必要
  - Word, Xlsx, PDF に含まれるテキスト情報を正しく Markdown 化できるか
  - NOTE: サトケンさんデモは 1 ページまるっと画像化して実装。複数ページでは利用不可能
- 多種多様なフォーマットのチェックリストから各チェック項目を構造化して抽出できるか要検証
- Context window に収まる分量に収まらない場合、どう分割するか
  - 書類はものによってはかなり分厚いが、全部 Input する必要はないと仮定して OK か（？）
-

プロジェクトの制約

- アルゴリズムの妥当性を判断するための、チェックリストが豊富に入手できない
  - これは AWS が LoB にリーチできていないことが原因
  - アセット展開後、アカウントチームからの FB によりブラッシュアップしていくことが求められる可能性が高い

プレスリリース

AWS、不動産業界向け AI ドキュメント適合性チェックシステム「BEACON」を発表

2025 年 6 月、東京 — Amazon Web Services（AWS）は本日、不動産・建設業界向けの AI ドキュメント適合性チェックシステム「BEACON (Building & Engineering Approval Compliance Navigator)」を発表しました。BEACON は、不動産業界で頻繁に発生する申請書類や契約書のチェック作業を大幅に効率化し、人手不足や長時間労働といった業界課題の解決に貢献します。
BEACON は申請書類に含まれる図面、テキスト、表などの非構造化データを自動的に分析し、規制やチェックリストに適合しているかを判断します。また、多種多様なフォーマットのチェックリストから各チェック項目を自動的に構造化して抽出する機能も備えており、業界特有の複雑なチェックリストにも対応可能です。これにより、従来は専門知識を持つ担当者が何時間もかけ、つきっきりで行っていた作業を、自動で完了することが可能になります。具体的には年間 10 万件以上の書類、2 万時間以上の工数を削減します。

AWS アカウントチーム責任者は次のように述べています。「BEACON は不動産業界という新たな市場セグメントへの参入機会を提供します。生成 AI の具体的なビジネス価値を示すことで、従来はクラウド移行に消極的だった企業とも対話を深めることができます。すでに複数のアカウントから高い関心を得ており、Azure 環境からの Winback の可能性も見えています。」生成 AI による書類審査ソリューションの導入を容易にします。

実際に BEACON を試用した大手不動産会社の書類審査担当者からは次のコメントが寄せられています。「申請書類の審査は私たちの業務の中で最も時間がかかり、ミスも起こりやすい作業でした。BEACON は技術に詳しくない私でもすぐに使いこなすことができ、チェックリストの作成から書類との照合まで自動化してくれるので、残業が大幅に減りました。何より、AI が判断に迷う部分を明示してくれるので、人間の判断が必要な箇所に集中できるようになりました。」

大手不動産会社の業務部門責任者は人材課題について次のように語ります。「慢性的な人手不足の中、書類審査業務は大きな負担となっていました。経験豊富な審査担当者の採用が難しく、育成にも時間がかかります。BEACON の導入により、限られた人的リソースを高付加価値業務に振り向けられるようになり、残業時間も 30%削減審査業務に要する時間が n%削減できました。特に、新人でも迅速に業務をこなせるようになったことは、当社の人材戦略上も大きな前進です。業界全体の高齢化が進む中、このようなテクノロジーは不可欠です。」

AWS 認定パートナーである建設システム開発株式会社の SI 部門マネージャーは次のように語ります。「BEACON は私たちのビジネス拡大の強力なツールです。このソリューションをベースに、顧客固有の業務フローや規制に合わせたカスタマイズを提供することで、新たな顧客層を開拓できています。オープンなアーキテクチャにより、既存の社内システムとの統合も容易で、お客様からの評価も非常に高いです。」

BEACON は GitHub で公開され、AWS パートナーネットワークのメンバーが自社のお客様に最適なカスタマイズを施すことができる設計になっています。

FAQ

- Q: BEACON の主なユースケースはなんですか？
  - A: 書類がチェックリストに適合しているかチェックします
    - 書類: 契約書、申請書（図面、テキスト、表含む）
  - A: 契約書の内容がチェックリストに適合しているかを確認します
  - A: 図面・テキスト
  - A: 申請書類（図面・テキスト・表）が、チェックリストに適合しているかを確認します
- Q: すべてのチェックリストをカバーしますか？
  - A: 複雑な依存関係はカバーしません。なぜなら業務効率化が目的であり、人手で（AI ができなかったところを）やる運用でも、目的を達成できるからです
- Q: 英語圏（外国語圏）でも利用できますか？
  - GitHub の README.md トップは日本語ですが、i18n 対応はします (Prototype チームの事情 + ECR TFC)
- Q: 想定ユーザーは誰ですか？またその人は何の目的で利用しますか？
  - 上記参照
- Q: PJ 成功はどのように測定しますか？
  - A: Sales contribution
    - A: Oppty キャンペーンコードの紐付けなどにより Revenue 合計額をトラックします
    - A: (TBD) パートナー経由で発生した Utility Revenue をトラッキングします
  - A: Thought leadership
    - A: 各種展示会 (re: Invent、AWS サミットの他、業界特化の展示会) で露出した回数をカウントします
    - A: PV 数（ブログ）、git clone 数
    - A: Tier1 / Tier2 事例の件数
- Q: 建築・不動産以外の業界にも利用できますか？
  - 使えれば使って OK ですが、そのユースケースへの特別対応はしません。
- Q. XX という機能を追加してほしいのですが、対応していただけますか？
  - アセットのコアコンセプトに沿っており、かつ案件規模や難易度に応じて決定しますので、お約束はできませんが、ものによっては対応します
- Q: 閉域対応したいです
  - A: SaaS 使っているお客様も多いです。必要であればパートナー紹介しますが、オープンだと無料で PoC できます

  - word, excel, pdf の 3 種類に加え、画像 (png, gif, jpg) n 種に対応します
- Q: データ食わせたら、精度がいまいちです。精度向上させる機能・方法はありますか (LLM ops)
  - A: まず、なんのデータを使い、具体的に何がうまくいっていないのかを教えてください。
  - A: 基本、アカウント担当にご相談ください

- Q: 導入する方法と、期間の目安、そして導入する前に準備すべきものを教えてください
  - A: AWS アカウントがあれば OK です。CloudShell から、非技術者でも簡単にデプロイ可能です。デプロイにかかる時間は 30 分ですが、放置すれば OK です
